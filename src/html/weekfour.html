Week Four IL Activity.

Task 1

I want my website to be as "clean" and pleasent to use as possible. The colour scheme I have initially went for is a pallette I picked from the Coolers website.
I felt the pallete was pleasent to look at, but after further research into to colour tones and the effect colour can have on projects, I feel like this colour scheme is too causal and more like a "blog" than an informative website. It doesn't evoke a sense of professionalism in the way I want it too. The websites colour pallete will most certainly change over time.

Orginal Pallette

Updated Pallette

Why the new pallete?

After finding more resources that are similar to my project topic, and identifying my target audience being 13-18 years old, I found that a brighter more "fun" pallete would suit the website more, providing eye catching visual and feeling a lot less "serious", albeit the topic is serious, this can often put users off engaging with the content.

Lab Work

Task 1

Task one required me to create my .css folder name it logically. I called it "mycss.css" and I linked my stylesheet in my HTML documents that is able to be viewed in my code via the GitHub repository.

Task 2

Task two required me to apply my CSS skills to the website and develop my style sheet. The result: what you see on this page!
I will develop much of the CSS as I progress in this project, as I mentioned before I feel like the original colour pallette doesn't suit my topic of choice, and needs to feel more professional and clean.
One of the important aspects of the CSS styling was to ensure each page has a consistent style across the board. This ensures that the user experience is fluid across the website and easy to navigate. The font I have picked is clear and easy to read in both the headers and paragraphs and by sticking to a dark font on a light background esures further readability.
